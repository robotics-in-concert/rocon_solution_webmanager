새로운 솔루션을 생성하고, 그 내부에 구성파일들을 생성합니다.

  LOCAL CREATE   CATKIN_CREATE_PKG   FileList    

Created Solution Files Description

새로운 솔루션을 생성하면 기본 구성파일로 아래와 같은 파일들이 생성됩니다.

  - package.xml : 솔루션에 대한 메타 정보 및 의존성 설정 파일
  - CMakeLists.txt : 설치 시 필요한 의존성 정보 검색을 위한 설정 파일

Step 1

아래에서 편집하고자 하는 솔루션을 선택하세요.

Step 2

선택한 솔루션에서 편집하고자 하는 구성파일을 선택하세요.

Step 3

아래 항목을 차례차례 입력하고 Modify 버튼을 누르세요.

1. 패키지명을 입력하세요.


2. 패키지 버전을 입력하세요.

3. 패키지에 대한 설명을 입력하세요.


4. Maintainer 정보를 입력하세요.



5. 라이센스(License)를 입력하세요.


6. URL 정보를 입력하세요.



7. 저작자(Author)를 입력하세요


8. Buildtool Depend를 입력하세요.

9. Run Depend를 입력하세요.


10. Export(concert_service)를 선택하세요.



입력값이 올바른지 확인 후 Modify 버튼을 누르세요.



Modify

1. cmake_minimum_required 정보를 확인하세요. 필요하면 수정하세요.


2. 프로젝트(project)를 입력하세요.

3. find_package 정보를 확인하세요. 필요하면 수정하세요.


4. catkin_package 를 확인하세요.

입력값이 올바른지 확인 후 Modify 버튼을 누르세요.



Modify

1. 서비스명을 입력하세요.


2. 서비스에 대한 설명을 입력하세요.

3. 저작자(Author)를 입력하세요.


4. 우선순위(Priority)를 입력하세요.

5. 런처 타입(Launcher Type)을 입력하세요.


6. 런처(Launcher)를 입력하세요.

7. 아이콘(Icon)을 선책하세요.


8. 상호작용(Interactions)을 입력하세요.

9. 파라미터(Parmeters)를 입력하세요.


입력값이 올바른지 확인 후 Modify 버튼을 누르세요.



Modify

1. 노드명(Node name)을 입력하세요.


2. 노드 패키지(pkg)와 타입(type)을 입력하세요.



3. 파라미터명(Param name)을 입력하세요.


4. 파라미터값을 입력하세요.

5. Include File을 선택하세요.


6. Argument명을 입력하세요.


7. Argument값을 입력하세요.

입력값이 올바른지 확인 후 Modify 버튼을 누르세요.



Modify
